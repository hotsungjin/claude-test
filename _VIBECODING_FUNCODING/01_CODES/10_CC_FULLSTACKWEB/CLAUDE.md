# AICOM

### 프로젝트 개요
- **목표**: 글 작성·댓글·검색 기능을 포함한 게시판 서비스 구축

### 기술 스택 (모놀리식 SSR)

- **백엔드 & SSR 프론트**  
  - Python 3.13 + FastAPI
  - 템플릿: **Jinja2**  
  - UI: Tailwind CSS 3.4 + **HTMX**(부분 갱신)  
- **인증·DB·파일 저장**: **Supabase**  
  - Auth·Storage·SQL은 Supabase API를 *서버 측(FastAPI)* 에서만 호출
  - **브라우저에는 Supabase JS SDK·키를 포함하지 않음**  
- **배포**: 도커컴포즈 (uvicorn 기반 실행 FastAPI 도커, nginx 도커로 구성)
  - 로컬에서도 도커컴포즈 명령으로 테스트 가능토록 구성

### 핵심 작업 원칙
- 모든 작업은 항상 ultra think 해서 버그없이 정확히 각 단계를 작업해주세요
- **테스트 환경 분리 없음**: 모든 테스트는 실제 환경(.env)에서만 수행합니다. 별도의 테스트 환경이나 .env.test 파일을 만들지 않습니다.
- 모든 작업은 
  1. **반드시 @PROGRESS.md 파일을 먼저 확인**하여 현재 프로젝트 상태와 구조를 파악
  2. TDD 기반으로 구현 계획을 세우고
  3. sub agents 로 병렬 실행 가능한 태스크를 분류하여,
  4. 도커컴포즈를 직접 실행하여, TDD 기반 API 테스트 및, puppeteer mcp 기반 사용자 테스트를 수행하고, 도커컴포즈 로그까지 확인하여
  5. 모든 버그를 수정합니다
- @ARCHITECTURE.md 파일의 구조 기반으로 구현해주세요
- **작업 전후 @PROGRESS.md 업데이트**: 중요한 변경사항, 새로운 설정, 구조 변경 등은 즉시 @PROGRESS.md에 반영

### 주요 기능
- 게시판 서비스에서는 여러 서브 게시판(공지사항, 뉴스레터 모음등)이 있고, 권한에 맞게 사용자는 각 게시판 활용 가능
- 인증 기능
  - 이메일 인증은 사용하지 않음(Supabase Auth (이메일/비밀번호, Confirm email OFF)) 
  - 세션: access_token + refresh_token → HttpOnly·Secure 쿠키 저장
  - FastAPI 미들웨어에서 JWT를 로컬 디코딩해 인증(별도 Supabase 호출 없음)
  - 사용자 정보: id(UUID), email, username, is_admin(Boolean), created_at만 저장
  - 단순화된 구조: users 테이블 하나로 모든 사용자 정보 관리 
- 사용자 기능 (경로: /)
  - **부분 갱신 방식**: 메인 페이지에서 게시판 선택 시 HTMX로 중앙 영역만 갱신
  - **페이지네이션**: 게시글 목록은 페이지 단위로 표시, 페이지 전환도 HTMX로 처리
  - 게시글 작성, 수정, 삭제
    - 리치 텍스트 에디터 지원 (Quill.js)
      - 볼드체, 제목(H3), 이텔릭체, 글자 색상, 글자 배경색상, 리스트
      - 이미지 붙여넣기 가능 (Base64 인코딩하여 저장)
  - 댓글 작성
  - 대댓글 작성 (2레벨)
  - 첨부 파일은 지원하지 않음
  - 검색: 전체 게시판 대상 (제목/내용 옵션)
- 관리자 기능 (경로: /admin)
  - 서브 게시판 생성, 수정, 삭제
  - 서브 게시판별 사용자 글쓰기, 보기 권한 부여
  - 사용자에게 관리자 권한 부여 가능
  - 초기 관리자 ID/PW는 admin@example.com / admin123
- 기타 페이지
  - 이용약관 (/terms)
  - 개인정보처리방침 (/privacy)
